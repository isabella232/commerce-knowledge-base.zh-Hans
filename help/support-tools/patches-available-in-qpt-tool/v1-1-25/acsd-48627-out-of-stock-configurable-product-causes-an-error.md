# ACSD-48627：缺货的可配置产品导致错误

ACSD-48627修补程序修复了在发送GraphQL请求以获取购物车详细信息时，缺货的可配置产品会导致错误的问题。 此修补程序在以下情况下可用： [[!DNL Quality Patches Tool (QPT)]](/help/announcements/adobe-commerce-announcements/magento-quality-patches-released-new-tool-to-self-serve-quality-patches.md) 已安装1.1.25。 修补程序ID为ACSD-48627。 请注意，该问题计划在Adobe Commerce 2.4.6中修复。

## 受影响的产品和版本

**该修补程序是为Adobe Commerce版本创建的：**

* Adobe Commerce（所有部署方法） 2.4.5-p1

**与Adobe Commerce版本兼容：**

* Adobe Commerce（所有部署方法） 2.4.5 - 2.4.5-p1

>[!NOTE]
>
>该修补程序可能适用于具有新版本的其他版本 [!DNL Quality Patches Tool] 版本发布。 要检查该修补程序是否与您的Adobe Commerce版本兼容，请更新 `magento/quality-patches` 包到最新版本，并检查 [[!DNL Quality Patches Tool]：搜索修补程序页面](https://experienceleague.adobe.com/tools/commerce-quality-patches/index.html). 使用修补程序ID作为搜索关键字来查找修补程序。

## 问题

缺货的可配置产品在发送GraphQL请求以获取购物车详细信息时会导致错误。

<u>重现问题的步骤</u>：

1. 创建客户帐户。
1. 将一些产品（包括可配置的产品）添加到购物车。
1. 转到管理员后端，通过将所有子产品数量设置为0来编辑可配置产品。
1. 由于所有子产品缺货，因此该可配置产品将缺货。
1. 查看 `catalog_product_index_price` 表格。 此产品的记录为空。
1. 发出GraphQL请求以获取客户令牌。

   ```GraphQL

       generateCustomerToken(
           email: "test@example.com"
           password: "xxxx"
           ) {
               token
               }
               }
   ```

1. 发出GraphQL请求以获取cartId。
